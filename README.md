# Moki (木几日记)

## 1. 品牌识别 (Brand Identity)

### 1.1 产品名称

- **英文名**：**Moki**
  - _发音_：/moʊki/
  - _寓意_：发音轻巧、灵动，象征无负担的记录体验。
- **中文名**：**木几日记**
  - _寓意_：取意“心中有一张干净的木几，安放生活的零碎”。
  - _意象_：温暖的木质纹理、阳光洒在桌面上的平静感、归属感。

### 1.2 核心 Slogan

- **中文**：“心中一隅，安放日常。”
- **英文**："Taste life twice." (致敬 Anaïs Nin: _"We write to taste life twice, in the moment and in retrospection."_)

---

## 2. 产品愿景 (Product Vision)

### 2.1 核心理念

- **简约 (Minimalist)**：拒绝繁杂的功能堆砌，回归记录本质。
- **温暖 (Warm)**：用视觉和交互传递温度，而非冷冰冰的工具感。
- **平静 (Calm)**：在这个喧嚣的世界里，提供一个安静的数字避难所。
- **低摩擦 (Frictionless)**：消除“写作焦虑”，让记录像呼吸一样自然。

### 2.2 目标体验

> "把‘记录’这件事情变得像发消息一样简单。不需要思考标题，不需要选择封面，只是记录当下。"

我们不希望用户因为“要写一篇完美的日记”而感到压力。打开 Moki，就像是在给未来的自己发一条微信，随时随地，只言片语皆是珍贵。

---

## 3. 设计美学 (Design Aesthetics)

### 3.1 视觉风格

- **色调**：
  - 主色调：米色 (Beige)、暖白 (Warm White)、纸张色。
  - 强调色：琥珀色 (Amber)、木棕色 (Wood Brown)、柔和的莫兰迪色系。
  - _参考意象：清晨的阳光、斑驳的树影、陈旧的书页、温润的木头。_
- **排版**：
  - 大留白，呼吸感强。
  - 字体优雅，易读性高，具有文学气质。
- **UI 元素**：
  - 卡片式设计（Card-based），承载每一个生活切片。
  - 极简图标，无多余装饰。

### 3.2 交互原则

- **即开即写**：打开 App 直接进入输入状态，无启动广告，无复杂的 Dashboard。
- **流式记录**：像聊天界面一样的 Timeline 视图，强调时间的连续性。
- **无压输入**：不强制要求标题、标签、天气等元数据（可自动获取或后补）。

### 3.3 设计哲学与细节洞察 (Design Philosophy & Insights)

> **"Make it feel like a letter, not a database."** (让它感觉像一封信，而不是一个数据库。)

- **不做视觉阻断**：

  - **标签处理**：坚决避免使用实心色块背景（Bad: 灰色胶囊块）。这会打破视觉流动的顺畅感，带来“异物感”。
  - **正确做法**：使用文字颜色区分（如钢笔圈画的感觉）或极细描边。让标签像是在信纸上用不同颜色的笔书写的，保留纸张的呼吸感。

- **做减法 (Subtraction)**：

  - 如果纯文字能表达清楚，就不要加图标。
  - 如果颜色能区分层级，就不要加背景块。
  - 每一个像素的增加都需要理由。

- **内容的尊严**：
  - 元数据（时间、位置）应该退后，内容（文字、图片）应该前置。
  - 不要让 UI 抢了用户记忆的风头。

## 4. 目标用户 (Target Audience)

- **高感性人群**：喜欢记录生活细节，对美有追求的用户。
- **极简主义者**：厌倦了复杂笔记软件（如 Notion/Obsidian）的用户。
- **日记新手**：想写日记但总是坚持不下来，因为传统日记压力太大的用户。
- **独立思考者**：需要一个私密空间与自己对话的人。

---

## 5. 核心功能 (MVP Scope)

1.  **快速记录**：类似 IM 的输入框，支持文字、图片。
2.  **时间轴展示**：按时间倒序排列的卡片流。
3.  **自动元数据**：自动记录时间、日期、天气（可选）。
4.  **回顾机制**：“那年今日”或随机漫游，帮助用户“品尝生活两次”。
5.  **隐私保护**：本地存储优先，iCloud 同步，FaceID 锁定。

---

## 6. 灵感来源

- **氛围**：_Komorebi_ (木漏れ日) - 阳光透过树叶缝隙漏下的光斑。
- **哲学**：记录当下，是为了在未来回味时再次拥有它。
